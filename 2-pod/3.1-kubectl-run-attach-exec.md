# kubectl run, kubectl attach e kubectl exec

## kubectl run

Comando usado para **criar e executar** um Pod no cluster Kubernetes de forma rápida.

### Sintaxe básica

```bash
kubectl run <nome-do-pod> --image=<imagem>
```

### Exemplos práticos

```bash
# Criar um pod simples com nginx
kubectl run nginx-pod --image=nginx

# Criar pod com porta exposta
kubectl run nginx-pod --image=nginx --port=80

# Criar pod com variáveis de ambiente
kubectl run app-pod --image=myapp --env="ENV=production"

# Criar pod com múltiplas variáveis
kubectl run app-pod --image=myapp --env="DB_HOST=mysql" --env="DB_PORT=3306"

# Criar pod com labels
kubectl run nginx-pod --image=nginx --labels="app=web,tier=frontend"

# Criar pod com limites de recursos
kubectl run nginx-pod --image=nginx --requests='cpu=100m,memory=256Mi' --limits='cpu=200m,memory=512Mi'

# Criar pod e expor como serviço
kubectl run nginx-pod --image=nginx --port=80 --expose

# Criar pod em namespace específico
kubectl run nginx-pod --image=nginx -n development

# Criar pod em modo interativo (e deletar ao sair)
kubectl run -it debug-pod --image=busybox --rm -- /bin/sh

# Criar pod com comando customizado
kubectl run busybox-pod --image=busybox -- sleep 3600

# Criar pod e executar comando específico
kubectl run curl-pod --image=curlimages/curl -- curl http://example.com
```

### Flags importantes

- `--image`: Imagem do container (obrigatório)
- `--port`: Porta que o container expõe
- `--env`: Define variáveis de ambiente
- `--labels`: Adiciona labels ao pod
- `--requests`: Define resource requests
- `--limits`: Define resource limits
- `--expose`: Cria um Service para o pod
- `--rm`: Remove o pod após término (útil para debug)
- `-it`: Modo interativo com TTY
- `--dry-run=client`: Simula criação sem executar
- `-o yaml`: Gera YAML do pod

### Casos de uso

- **Testes rápidos**: Criar pods temporários para validações
- **Debug**: Pods de diagnóstico com ferramentas específicas
- **Prototipagem**: Testar imagens antes de criar manifests
- **Troubleshooting**: Pods utilitários para testar conectividade
- **Desenvolvimento**: Execução rápida durante desenvolvimento

### Exemplos de debug

```bash
# Pod temporário para testes de rede
kubectl run netshoot --rm -it --image=nicolaka/netshoot -- /bin/bash

# Pod para testar DNS
kubectl run dnsutils --rm -it --image=tutum/dnsutils -- /bin/sh

# Pod para fazer requisições HTTP
kubectl run curl --rm -it --image=curlimages/curl -- sh

# Pod busybox para debug geral
kubectl run busybox --rm -it --image=busybox -- sh
```

### Gerar manifests YAML

```bash
# Gerar YAML sem criar o pod
kubectl run nginx-pod --image=nginx --dry-run=client -o yaml

# Salvar YAML em arquivo
kubectl run nginx-pod --image=nginx --dry-run=client -o yaml > pod.yaml

# Gerar YAML com configurações complexas
kubectl run app-pod --image=myapp --port=8080 --env="ENV=prod" --dry-run=client -o yaml > app-pod.yaml
```

## kubectl exec

Comando usado para **executar comandos** dentro de um container em execução.

### Sintaxe básica

```bash
kubectl exec <nome-do-pod> -- <comando>
```

### Exemplos práticos

```bash
# Executar comando simples
kubectl exec nginx-pod -- ls /usr/share/nginx/html

# Executar comando com argumentos
kubectl exec nginx-pod -- cat /etc/nginx/nginx.conf

# Modo interativo com shell
kubectl exec -it nginx-pod -- /bin/bash
kubectl exec -it nginx-pod -- /bin/sh

# Executar em container específico (pod multi-container)
kubectl exec -it my-pod -c container-name -- /bin/bash

# Executar em namespace específico
kubectl exec -it nginx-pod -n production -- /bin/bash

# Ver variáveis de ambiente
kubectl exec nginx-pod -- env

# Verificar processos
kubectl exec nginx-pod -- ps aux

# Testar conectividade
kubectl exec -it app-pod -- curl http://service:8080

# Verificar DNS
kubectl exec -it app-pod -- nslookup kubernetes.default

# Ver espaço em disco
kubectl exec app-pod -- df -h
```

### Flags importantes

- `-i` ou `--stdin`: Modo interativo
- `-t` ou `--tty`: Aloca terminal TTY
- `-c` ou `--container`: Especifica container
- `-n` ou `--namespace`: Define namespace

### Casos de uso

- **Debug**: Investigar problemas dentro do container
- **Inspeção**: Verificar arquivos e configurações
- **Testes**: Validar conectividade e DNS
- **Manutenção**: Executar comandos administrativos
- **Troubleshooting**: Diagnosticar problemas

## kubectl attach

Comando usado para **conectar-se ao processo principal** (PID 1) de um container em execução.

### Sintaxe básica

```bash
kubectl attach <nome-do-pod>
```

### Exemplos práticos

```bash
# Anexar ao processo principal
kubectl attach nginx-pod

# Modo interativo
kubectl attach -it nginx-pod

# Anexar a container específico
kubectl attach -it my-pod -c container-name

# Anexar em namespace específico
kubectl attach -it nginx-pod -n production
```

### Flags importantes

- `-i` ou `--stdin`: Passa stdin para o container
- `-t` ou `--tty`: Stdin é um TTY
- `-c` ou `--container`: Especifica container

### Casos de uso

- **Aplicações interativas**: Conectar a apps que esperam entrada
- **Debug de processos**: Ver saída do processo principal em tempo real
- **Monitoramento**: Acompanhar stdout/stderr
- **Sessões interativas**: Conectar a shells em execução

### Importante

- Conecta apenas ao **processo principal** (PID 1)
- Não inicia novo processo como o `exec`
- Para sair sem matar o processo: `Ctrl+P` seguido de `Ctrl+Q`

## Comparação entre os três comandos

| Comando | Função | Quando usar |
|---------|--------|-------------|
| **kubectl run** | Cria e executa um novo Pod | Criar pods rapidamente para testes ou debug |
| **kubectl exec** | Executa novo processo no container | Debug geral, executar comandos, abrir shell |
| **kubectl attach** | Conecta ao processo principal existente | Ver saída do processo principal, apps interativas |

## Fluxo de trabalho típico

### 1. Criar pod de debug

```bash
# Criar pod temporário para troubleshooting
kubectl run debug-pod --rm -it --image=busybox -- sh
```

### 2. Criar pod persistente

```bash
# Criar pod que permanece em execução
kubectl run nginx-pod --image=nginx --port=80
```

### 3. Executar comandos no pod

```bash
# Abrir shell no pod criado
kubectl exec -it nginx-pod -- /bin/bash

# Executar comando específico
kubectl exec nginx-pod -- curl localhost:80
```

### 4. Anexar ao processo (se necessário)

```bash
# Conectar ao processo principal
kubectl attach -it debug-pod
```

## Exemplos práticos combinados

### Cenário 1: Debug de conectividade de rede

```bash
# 1. Criar pod de debug
kubectl run netdebug --rm -it --image=nicolaka/netshoot -- /bin/bash

# 2. Dentro do pod, testar conectividade
curl http://service-name:8080
nslookup service-name
ping pod-ip
```

### Cenário 2: Inspecionar aplicação em execução

```bash
# 1. Criar pod da aplicação
kubectl run myapp --image=myapp:v1 --port=8080

# 2. Verificar se está rodando
kubectl get pods

# 3. Executar comandos de debug
kubectl exec -it myapp -- /bin/bash

# 4. Dentro do container
ps aux
netstat -tulpn
curl localhost:8080/health
```

### Cenário 3: Teste rápido de imagem

```bash
# 1. Criar pod temporário
kubectl run test-app --rm -it --image=myapp:latest -- /bin/sh

# 2. Testar aplicação interativamente
# Pod é deletado automaticamente ao sair
```

### Cenário 4: Debug de DNS

```bash
# 1. Criar pod com ferramentas DNS
kubectl run dnstest --rm -it --image=tutum/dnsutils -- /bin/sh

# 2. Testar resolução DNS
nslookup kubernetes.default
nslookup service-name.namespace.svc.cluster.local
dig service-name
```

## Dicas práticas

### kubectl run
- Use `--rm -it` para pods temporários de debug
- Use `--dry-run=client -o yaml` para gerar manifests
- Combine com `--expose` para criar Service automaticamente
- Para produção, prefira usar Deployments ao invés de pods diretos

### kubectl exec
- Sempre use `-it` para sessões interativas
- Use `--` para separar flags do kubectl dos argumentos do comando
- Verifique qual shell está disponível (`/bin/bash` ou `/bin/sh`)
- Em pods multi-container, sempre especifique `-c container-name`

### kubectl attach
- Menos usado que `exec` no dia a dia
- Útil para containers de debug temporários
- Para apenas ver logs, use `kubectl logs -f` ao invés de attach
- `Ctrl+P` + `Ctrl+Q` para sair sem matar o processo

### Atalhos úteis

```bash
# Alias para facilitar
alias k='kubectl'
alias krun='kubectl run'
alias kexec='kubectl exec -it'
alias kdebug='kubectl run debug --rm -it --image=busybox -- sh'

# Uso
k get pods
krun nginx --image=nginx
kexec nginx-pod -- /bin/bash
kdebug
```
